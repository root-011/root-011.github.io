HTTP协议

HTTP （超文本传输协议）是一种应用层的协议[直接提供用户访问资源] ， 是一种客户端—服务器（client-server）协议

HTTP协议是无状态的：每个请求都是独立的，服务器不会记住之前的请求信息。

Cokkie存储在客户端，可设长期有效

Session（会议）存储在服务器端，会话结束失效

  • API（应用程序编程接口）：定义客户端与服务器之间的交互方式。
  • 常见HTTP方法
    • GET：获取资源
    • POST：创建资源或提交数据
    • PUT：更新资源
    • DELETE：删除资源
    • PATCH：部分更新资源
  • 状态码
    • 1xx：信息响应
    • 2xx：成功（如200 OK）
    • 3xx：重定向
    • 4xx：客户端错误（如404 Not Found）
    • 5xx：服务器错误（如500 Internal Server Error）

URL

【协议】**://【域名:端口】/【路径】?【查询参数】#**【锚点】

  • 协议、域名必需
  • 查询参数： 传递键值对数据（用&分隔）
  • 锚点： 页面内跳转位置（不发送给服务器）

https://www.example.com/items/?q=hat&price_range=10-100#recommend

http消息

常见的请求方式（ HTTP 方法 ）

  1. GET： 只被用于获取数据
  2. POST：提交数据给服务器 （创建或修改资源，或者产生要返回的临时文件）
  3. PUT：更新资源
  4. DELETE：删除资源
  5. HEAD：获取响应头
  6. OPTIONS：查询服务器支持的请求方法
  7. PATCH：部分更新资源
  8. TRACE：回显请求，用于测试
  9. CONNECT：建立隧道连接
GET POST
用途 请求数据（获取资源） 提交数据（创建或更新资源，返回文件）
参数传递位置 附加在 URL 中，作为查询字符串 放在请求体中
数据长度限制 受 URL 长度限制（通常为 2048 个字符） 无限制，适合传输大量数据。
安全性 参数暴露在 URL 中，不适合传输敏感信息 参数在请求体中，更安全
缓存 可以被浏览器缓存 默认不会被缓存
幂等性 幂等 非幂等
参数可见性 在 URL 中可见 不可见
应用 分享链接或获取数据 提交敏感或大量数据

  • GET传参

    参数附加在 URL 的查询字符串中（协议**://主机名:端口/路径?查询参数#**片段）

    格式：?key1=value1&key2=value2

  • POST传参

    放在请求体中

    表单格式：key1=value1&key2=value2

    JSON 格式：{"key1": "value1", "key2": "value2"}

HTTP请求与响应格式

请求结构

  1. 起始行： 方法+路径+协议版本 POST /login HTTP/1.1
  2. 请求头： 元数据（键值对）
  3. 空行
  4. 请求体：传输的数据

响应结构

  1. 状态行： 协议+状态码+描述 HTTP/1.1 200 OK
  2. 响应头： 元数据（键值对）
  3. 空行
  4. 响应体： 返回的实际内容
